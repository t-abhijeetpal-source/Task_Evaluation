## 5. Data Model Summary

### Core entities

- **`personal_details`** â€” cached logged-in user profile (`userId` PK). Evidence: `EquityDatabase.kt` entity list + schema v19. **Only PII table.**

- **Stock/search cache cluster** â€” `recent_search`, `recently_viewed`, `popular_search`, `most_invested_stocks` store denormalized instrument snapshots keyed by `stock_id` and/or auto `id`. Evidence: identical column shapes in schema v19 `createSql`.

- **Home dashboard cache** â€” `portfolio_details` and `common_details` are type-keyed JSON/value blobs for dashboard widgets.

- **PnL cache** â€” `equity_realised_*` and `fno_realised_*` tables cache realised P&L summary/detail rows from backend.

- **Logging DB** â€” `api_failure_logging`, `api_response_time`, `whitelist_url_tab` for diagnostics (separate SQLite file `logging`).

### Relationship hotspots

- **Stock identity columns** (`stock_id`, `isin`, `security_id`, `symbol`) appear across 5+ equity tables without FK enforcement â€” application-layer joins only.

- **PnL detail/summary pairs** share `isin` + `scriptName` shape but no parent-child FK.

- **Chart config lookup tables** (7 tables with `label`/`value` pattern) are independent reference data.

### Potential issues

1. **No foreign keys anywhere** â€” orphan rows possible; referential integrity is not enforced at DB level (typical mobile cache pattern).

2. **Denormalized stock snapshots** â€” same instrument metadata duplicated across search/viewed/popular tables; updates can drift.

3. **Mixed PK strategies** â€” some tables use natural keys (`stock_id`, `userId`, `type`, `screen_name`) while others use auto-increment `id`; complicates cross-table linking.

4. **`mtf_scrips.scripId` vs `stock_id`** â€” naming mismatch; relationship to search tables is INFERRED only.

5. **JSON-in-TEXT columns** â€” `equity_config` order-type fields (via `Converters`), `common_details.data` store serialized structures; schema does not document JSON shape.

6. **No soft-delete columns** â€” tables use hard replace/delete patterns (no universal `deleted_at`).

7. **Schema version gap** â€” exported versions jump (e.g. no `15.json` in `schemas/`); current head is v19 for equity, v7 for logging.
